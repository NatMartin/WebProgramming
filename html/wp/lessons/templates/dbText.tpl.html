DataBases

Databases contain data. Technically, a ledger book or a spreadsheet are databases, but usually, when databases refer to computer supported databases. On the computer, a Database Management System (DBMS) manages the data. The DBMS lets user create, read, update and delete (CRUD) the data it stores. It also makes sure that when multiple users update the same records reasonable things happen.

Relational Databases

Relational Databases were developed in the 1970's. They represent data as sets of relations, or tables. Technically, a relation is a mathematical object defined as a set of tuples, where a tuple is an ordered set of objects. A functions is a type of relation in which a subset of object in the tuple, uniquely determines all the rest of the objects: that is, any particular input to the function return a unique value. As a simplification, think of a relation as a table of values. Each row in the table is a tuple. The order of the rows is unimportant.

Most databases used in industry is relational databases.

The most common DBMS's use SQL (pronouced "sequel" standing for Structured Query Language). SQL uses a relational model where data is structured in tables where each row in the table represents a different object in the real world.
