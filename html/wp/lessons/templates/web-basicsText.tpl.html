The Internet and the World Wide Web

The Internet and the World Wide Web differ in the same way an aircraft carrier and an aircraft differ. The Internet carries the World Wide Web, but it carries a lot of other things too. The Internet is based on a protocol that supports the connection of multiple networks. These networks move the information. The World Wide Web is a protocol for requesting information over the interconnected networks and for displaying that information when it arrives.

Client Server Model

Both the Internet and the World Wide Web use the client-server model.

A client-server model includes two programs: a client and a program. The client program requests information from the server program; the server program delivers the information. Though conceptually simple, client server programs are somewhat complicated: the server program must provide a means of contacting it. When a client contacts it, the server must switch communication to another channel to free the channel on which other program are distending and start a new thread to handle the communication with the current client.

The client-server model separates concerns well. The server performs a simple computation and returns the result of that computation to the client. The client need know nothing about how the server computes the result; the server need know nothing about what the client does with the result.

Clients communicate with servers in one of two ways synchronous and asynchronous. When a client communicates with the server synchronously, it request a result from the client, then waits until the client returns it. When a client communicates asynchronously, it continues working while it waits for the result. Programs that communicate synchronously are easier to write; synchronous calls are just like function calls. Asynchronous programs are more difficult to write. The client must provide a way to access the results once they are returned.

The Internet

The Internet is a set of interlocking protocols. It grew out of the ARPAnet, which was active in the 1970's and 1980's. The ARPAnet was a US Military network that grew into a broad international scientific network. It was replaced by the Internet (a civilian network) under the influence of Al Gore's Congressional Committee.

The IETF (Internet Engineering Task Force) manages in Internet Protocols as a set of RFC's (Request for Comments). It has no governing powers, rather, it publishes standards that anyone may implement. However, unless the protocols are implemented exactly according to the IETF standards, the implementer's system will not connect to the internet. The inter-connectivity of the Internet governs it.

The Internet uses two protocols, TCP (Transmission Control Protocol) and IP (Internet Protocol), which are usually shortened to TCP/IP.

Packet Switching

IP specifies how computers share data. It is a packet switching protocol, which means any communication is broken into fixed sized chunks of data. IP specified the format of IP addresses such as familiar four number address for IPv4 (e.g. 192.168.0.1). These addresses are currently being replace by a IPv6 (there is no IPv5).

The addresses specify how to find the computer on the network. Each network has an designated internet address, which is associated with the network's router. The router then translate the external address using NAT (Network Address Translation) into an address that the internet network can use.

The traditional Internet addresses (e.g. www.wikipedia.com) are translated into numerical IP address using a (DNS) Domain Name Server, which accepts the named address and returns the corresponding IP address. These names are coordinated in a distributed fashion whereby each DNS has other DNS servers that it contacts for updates. This way changing the name on a single DNS can propagate to the entire network.

IP uses packet switching. Unlike a telephone, which works by circuit switching, the Internet protocol specifies that the sending computer break messages into packets and address each of these packets to the receiving computer. The receiving computer then puts the packets back together based on sequence numbers the sending computer provided. Each packet has a payload consisting of the data to be sent and a header that contains the destination address, the source address, and other meta-data needed to put the message back together. Each packet is sent out to the network's router which forwards it to other routers until the packet reaches its target.

TCP is built on top of IP. It takes the data stream being sent, breaks it into chunks adding meta-data that will allow it to reconstitute the message on the other side. It then send each of the chunks of data to the receiving computer. The TCP program on the receiving computer then captures the packets coming in and reconstitutes them into the stream that was sent. In this way, TCP simulates a circuit switching network over a packet switching network.

The World Wide Web

Like the Internet, the World Wide Web (WWW) is a set of protocols the implements a client-server model. The Web uses HTTP (Hypertext Transmission Protocol) and HTML (Hypertext Markup Language). HTTP specifies how computer communicate with each other over the Web, and HTML specifies what they send each other.

HTTP

HTTP defines a request-response protocol for the Web's client-server model. That is, it specifies how the client sends a request to a server and how the server responds to that request. It describes nine possible commands of which the most common are "GET", "PUT" and "POST". The client sends a GET command with a file specification and the server returns a header, and the data associated with the file if it can find the file. If it cannot, it returns a header with a "404" error. The client sends a PUT or a POST with data as well as a file specification. When the server receives the PUT or POST, it processes the data that comes with as specified in the file associated with the command.

HTTP covers a well defined and relatively simple space. It says how to send and retrieve documents over the Web. It simplicity, coupled with the difficulty of retooling every Web server and rewriting every Web browser has given it remarkable stability. HTTP 1.0 was released in 1996, followed later that same year by HTTP 1.1 containing some tweaks to the standard. Since then, the only update has been a reorganization of the RFC into five RFCs in 2007. We still use HTTP 1.1.

HTML

HTML is the format of the files that the server returns to the browser. For the most part, it indicates what kind of information is contained in the link. For example, an "H1" tag specifies that the data contain is the top level heading of the document. What made HTML important is the anchor tag "A". This tag treats its contents as a hyperlink. When the user clicks on the content of the tag, the browser issues a GET command for the file specified in the href value.

HTML has a livlier history than does HTTP because it specifies the format for documents, a less easily defined concept that fetching information. Moreover, instead of being implemented by servers, HTML is implemented by browsers, which are produced by a collection of commercial and non-commercial companies. These companies compete on the features they provide and, without changes to HTML, it is difficult to provide novel features. The desire to add new tags to HTML balances again the need to have people use these new tags. Since people use many different browsers, document that have tags supported by only one browser will display incorrectly in all of the other browsers providing a disincentive for authors to use these new tags. In the early days of the Web, Microsoft Internet Explorer dominated the browser market allowing Microsoft to define standard behavior. Now, however, enough people use other browsers to give value to writing according to the standard as defined by official standards.

As an unfortunate legacy of the time when it dominated the browser market, Internet Explorer still supports the official standard differently from all other browsers complicating the lives of Web authors. When Web browsers request a document, they indicate what kind of browser they are. Sometimes Web authors use this information to display different information depending on the browser requesting it. More commonly now, Web authors use libraries that manage the differences between browsers.
