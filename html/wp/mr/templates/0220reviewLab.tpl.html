Manual vs Automatic testing and End-to-end vs Unit testing are two orthogonal descriptions of testing. That is there is Manual End-to-end testing, Manual Unit testing, Automated End-to-end testing, and Automated Unit testing. All four are useful in different areas. The following describe properties of the tests.

Manual vs Automatic Testing

  • Manual
    1. Human follows steps; views answer
    2. Steps and observations must be clear
    3. Allows for observation and interpretation
    4. Costly
  • Automated
    1. Program executes steps; performs test
    2. Program executes steps; performs test
    3. Cheap

Unit vs End-to-end

  • Unit
    1. Programmer tests own code
    2. Does not catch intergration errors
    3. Automation helps with regressions errors
  • Automated
    1. Tests system behavior
    2. Does not indentify root cause
    3. Automation lets programmers catch regression errors
