In this lesson I will go through the development of the first program in the W3Schools JavaScript lessons.

Before we do this we will need to download two pictures from the server. To do this we will using wget to get the file from the server and tar to extract the files from the tar ball.

  1. cd ~/src/wp/html
  2. wget 172.16.24.22/img.tar
  3. tar -xvf img.tar
  4. When you do an ls, you should see the file img.tar and the directory img. The directory should have the two files we need.

First, typing in the following code. This simple HTML should take about five minutes. You may write less, but do not write more. Save the code in ~/src/wp/html/lightbulb.html

<html> <head> <meta charset=”utf-8”> <title>Lightbulb example</title> </head> <body> <h1>JavaScript Changes Images</h1> </body> </html>

We will test the code by starting the server and looking at the code in the browser. If the server is not running, start it. Then point the browser to localhost:30000/lightbulb.html.. Do you see the title and the header? If not, fix the code before moving on.

This little program performs a series of tests. By loading it in your server, you test that the server is running and that the page is loading properly.

Once you have it running, save it using git. Next, add a little bit more code.

<html> <head> <meta charset=”utf-8”> <title>Lightbulb example</title> </head> <body> <h1>JavaScript Changes Images</h1> <img src="img/pic_bulboff.gif"> <p> Click the light bulb to turn on/off the light. </p> </body> </html>

Refresh the your broswer window to make sure it looks right. If it does, add a little more. Do you see the picture of the lightbulb? If so, you are ready to go on. Possible error are that the light bulb is not in the right directory. It needs to be in ~/src/wp/html/img. Or you might be asking for it from the wrong directory. It needs to be a relative path "img/pic_bulboff.gif". Or you may have left off the 'gif'.

This code still does not add any JavaScript. Instead it test that the code you are editing is the code that is running in your server. It also shows that the image is in the right directory to be loaded. Again save the code using git before moving on to another small change.

<html> <head> <meta charset=”utf-8”> <title>Lightbulb example</title> </head> <body> <h1>JavaScript Changes Images</h1> <img onclick="changeImage()" src="img/pic_bulboff.gif"> <p> Click the light bulb to turn on/off the light. </p> <script type="text/javascript"> function changeImage() { document.write("function called"); } </script>> </body> </html>

What happens when you click on light bulb? Does the page go away and you see "function called". If so, the program worked.

Here we have tested that we can associated a function with an action on the image, and that the function will be called when we do the action. When it works, add it to git and continue

<html> <head> <meta charset=”utf-8”> <title>Lightbulb example</title> </head> <body> <h1>JavaScript Changes Images</h1> <img id="bulbImg" onclick="changeImage()" src="img/pic_bulboff.gif"> <p> Click the light bulb to turn on/off the light. </p> <script type="text/javascript"> function changeImage() { var image = document.getElementById('bulbImg'); image.src = "img/pic_bulbon.gif; } </script>> </body> </html>

Now when you click on the image, does the bulb turn on? If so, now change the logic so that it turns the bulb off too.

We have tested that the function will change the image. Add it to git and finish the code.

<html> <head> <meta charset=”utf-8”> <title>Lightbulb example</title> </head> <body> <h1>JavaScript Changes Images</h1> <img id="bulbImg" onclick="changeImage()" src="img/pic_bulboff.gif"> <p> Click the light bulb to turn on/off the light. </p> <script type="text/javascript"> function changeImage() { var image = document.getElementById('bulbImg'); if (image.src.match(“bulboff”)) { bulbImg.src = "img/pic_bulbon.gif" } else { bulbImg.src = “img/pic_bulboff.gif” } } </script>> </body> </html>
