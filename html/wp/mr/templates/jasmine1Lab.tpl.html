Dowloading Jasmine

Jasmine is a JavaScript program, which means that we can run it in out web browser. This makes it a great testing framework for the web. The tests we run are running in the JavaScript interpreter that the broswer will run our web pages.

Installing Jasmine involves getting the JavaScript files and installing them on our web page. We then include those pages on a web page with the JavaScript programs we have written and the specification of those JavaScript programs. To use Jasmine you need to write tests in JavaScript as well as the programs. This may seem like extra work, but in saves more work in the long run.

To get Jasmine we will first load it onto our computer from the server at 172.16.24.22 (I put it there over the weekend). You will then unzip it and put it in the appriate diretory

  1. cd ~/src/wp
  2. mkdir jasmine
  3. cd jasmine
  4. wget 172.16.24.22/jasmine.zip
  5. unzip jasmine.zip
  6. Make sure your server is running, then open a Web browser and navigate to localhost:3000/jasmine/SpecRunner.html.

Looking at what we have

Open ~/src/wp/jasmine/SpecRunner.html. You will see an odd looking html page. I had only a head; no body. The head loads a number of CSS and JavaScript files. The head looks like this:

<head> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8"> <title>Jasmine Spec Runner v2.0.2</title> <link rel="shortcut icon" type="image/png" href="lib/jasmine-2.0.2/jasmine_favicon.png"> <link rel="stylesheet" type="text/css" href="lib/jasmine-2.0.2/jasmine.css"> <script type="text/javascript" src="lib/jasmine-2.0.2/jasmine.js"></script> <script type="text/javascript" src="lib/jasmine-2.0.2/jasmine-html.js"></script> <script type="text/javascript" src="lib/jasmine-2.0.2/boot.js"></script> <!-- include source files here... --> <script type="text/javascript" src="src/Player.js"></script> <script type="text/javascript" src="src/Song.js"></script> <!-- include spec files here... --> <script type="text/javascript" src="spec/SpecHelper.js"></script> <script type="text/javascript" src="spec/PlayerSpec.js"></script> </head>

The lines that are surrounded by script tags are JavaScript programs. The top three jasmine.js, jasmine-html.js and boot.js are the JavaScript code that runs Jasmine. The next two Player.js, and Song.js are the application being tested by the example program. The last two, SpecHelper.js and PlayerSpec.js are the specification files. The last two contating the Jasmine code.

For example, a short snippet of the PlaySpec code shows the sturcture of Jasmine programs.

describe("Player", function() { var player; var song; beforeEach(function() { player = new Player(); song = new Song(); }); it("should be able to play a Song", function() { player.play(song); expect(player.currentlyPlayingSong).toEqual(song); });

Each Jasmine spec describes some piece of the program, usually a class. Here, we are describing the Player class. The description takes the name of the thing that is being descibed and a function as a parameter. The function can set up variables and functions just like any other JavaScript function. The most important are the "it" functions. These functions are the tests. Here there is one test, which takes a string as it's first argument and a second as its second argument. The string describes what the function is testing. Here the Player object "should be able to play a Song". To see if it can play a song, it runs the program. The expect functions in either succeed or fail. If all of the expect functions succeed, the funciton succeeds and the test is passed. If one of them fails, it notes that that function did not pass. For example, here if the song currently being playsed is not the song the player was told to play, the test of playing a song fails.
