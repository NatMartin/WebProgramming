Continuing TDD on the Server

In the last lesson we were working on the server using TDD. However, most of the development was in the specification. This indicates a useful application of TDD, namely it is helpful in exploring new frameworks. However, we have not yet made much progress on our story. As you recall, the story is: "As a blog writer, I want to be able to save my posts so that I do not have to retype them."

We saw Test Driven development:

  1. Write a failing test. If it does not fail, we do not need to write any code, so we can go on to write a new test until we get one that fails.
  2. Write the smallest program that will make the test succeed. By the smallest program, we really mean the smallest program. For example we can write a multiply funciton that simply retruns ten if the test is to multiply five by two. We will add more tests until we have a real multply function. Writing the program should not take more than five to ten minutes.
  3. Refactor the code so that it is easy to understand. This means first, making the code look good with proper indentation and expressive function and variable names. Second, look for any code that is repeated. If you find any pull it out into its own function. Third, make sure that each function only do one thing and that the one thing they do is clearly expressed by its name. Do the same for class, methods and properties. Refactoring is the step that turns student code into professional code.

Today we will continue with the development of the story. The first thing we need to do break the story down into tasks. Some of the tasks are done on the server and some are done on the client (i.e., browser).

  1. When first creating the page, read the previous blog posts from the array on the disk to create the page. (Server)
  2. Form the array of blog posts into a web page. (Server)
  3. Display the web page the server has sent on the browser. (Client)
  4. Send the information from the browser to the server. (Client)
  5. Capture the information sent from the browser at the server. (Server)
  6. Put the captured information sent from the browser in an array on the server. (Server)
  7. Save the array on disk on the server. (Server)
  8. Format the information in the array on the server into the web page to be sent back to the browser. (Server)
Since only one of the tasks needs to be done on the client, lets tackle the server side first.
